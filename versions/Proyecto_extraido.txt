Proyecto: SecureVault API
(un gestor de credenciales/secretos seguros como microservicios)
¿Por qué se selecciona para la materia?
Ciberseguridad: Encriptación AES-256 en reposo, JWT + OAuth2 para auth, validación de inputs (OWASP), rate-limiting, logging de auditoría, secrets management (no hardcodeados), HTTPS/TLS.
Cloud Security: Simula un entorno cloud con variables de entorno (Docker secrets), políticas de IAM básicas y encriptación de datos.
DevOps: Docker + Docker Compose (multi-contenedor), CI/CD básico con GitHub Actions, Dockerfile optimizados, .dockerignore, healthchecks.
Microservicios: 2 servicios independientes (puedes escalar a 3 si quieres más puntos).
Funcional y publicable: API REST consumible por cualquier frontend/Postman, se publica en Docker Hub y se corre con un solo comando.
Sencillo y rápido: Todo en Python (FastAPI) o Node.js (Express) – elige lo que ya domines. Base de datos ligera (SQLite o PostgreSQL).

Opción A – Recomendada (Python + FastAPI)
Más moderna, ligera, excelente documentación automática (Swagger), y muy valorada en entornos DevSecOps/Cloud 2025–2026.
Tecnologías principales:
Backend: Python 3.11+ + FastAPI + Uvicorn
Autenticación: JWT (PyJWT) + OAuth2 Password Bearer
Encriptación: cryptography (AES-256-GCM con clave derivada por usuario via PBKDF2)
Base de datos: PostgreSQL (o SQLite para simplicidad inicial)
Rate limiting: slowapi
Logging: structlog o logging JSON
Docker: multi-stage build, non-root user, HEALTHCHECK
Orquestación: Docker Compose (auth + vault + db)

text
